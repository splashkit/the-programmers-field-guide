A useful way to design the classes in our programs is to think about what *role* objects of that class will have, and the *responsibilities* associated with that role.

Watch this video to:

- Get starting with thinking in terms of roles and responsibilities.
- See how UML diagrams can be used to communicate the design of a class.

When you have gone through this video you should be able to start using the ideas of roles and responsibilities to design classes.
